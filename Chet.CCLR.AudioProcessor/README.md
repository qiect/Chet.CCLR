# 道德经音频拆解处理方案

基于 `音频拆解设计_v1.md` 的完整实现方案，专门针对道德经音频进行精细化处理。

## 方案概述

本方案实现了从单一长音频文件到多个句子级音频片段的自动化处理流程，具体包括：

1. 文本自动分句处理
2. 音频时间轴标注
3. 音频精确切割
4. 数据库脚本生成

## 技术架构

### 1. 文本分句模块
- 基于正则表达式的中文标点符号识别
- 支持句号、感叹号、问号、分号等断句
- 特殊处理章节标题与正文内容分离
- 生成471个句子片段（道德经全文）

### 2. 音频处理模块
- 集成FFmpeg音频切割功能
- 支持多种音频格式输入（M4A, MP3等）
- 精确到毫秒的时间轴控制
- 输出标准化的MP3格式音频

### 3. 数据库集成模块
- 生成符合 `classic_sentence` 表结构的SQL脚本
- 自动关联句子内容与音频文件
- 支持章节、排序、发布状态等字段

## 使用流程

### 第一步：准备阶段
1. 确保源文件存在：
   - 文本文件：`Chet.CCLR.Source\道德经\道德经_王弼本.txt`
   - 音频文件：`Chet.CCLR.Source\道德经\道德经_全文朗读_王弼本.m4a`

### 第二步：时间轴标注
1. 使用 Audacity 打开音频文件
2. 按句子边界添加标签（Ctrl+B）
3. 导出标签为文本文件（labels.txt）
4. 将标签文件放入 `output` 目录

### 第三步：音频切割
1. 运行 `AudioProcessor.exe`
2. 选择"执行音频切割"
3. 系统自动按标签时间轴切割音频
4. 生成对应数量的音频片段

### 第四步：数据入库
1. 检查生成的 `insert_script.sql`
2. 根据实际章节信息修改 `chapter_id`
3. 执行SQL脚本导入数据库
4. 验证音频文件路径正确性

## 扩展性考虑

1. **多书籍支持**：可通过配置文件扩展到其他经典著作
2. **多语言支持**：分句算法可适配不同语言标点规则
3. **云存储集成**：音频文件路径可配置为云存储地址
4. **批处理模式**：支持命令行参数进行无人值守处理
