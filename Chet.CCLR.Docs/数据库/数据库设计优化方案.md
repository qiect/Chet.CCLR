# 初始版本数据库设计优化方案

## 1. 用户表优化 (`user`)

```sql
CREATE TABLE user (
  id               VARCHAR(32) PRIMARY KEY,
  wx_openid        VARCHAR(64) NOT NULL UNIQUE,
  nickname         VARCHAR(50),
  avatar_url       TEXT,
  gender           TINYINT DEFAULT 0,  -- 0未知 1男 2女
  country          VARCHAR(20),
  province         VARCHAR(20),
  city             VARCHAR(20),
  language         VARCHAR(10) DEFAULT 'zh-CN',
  subscribe_time   TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  last_login_time  TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  status           TINYINT DEFAULT 1,  -- 1正常 0禁用
  created_at       TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at       TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);

-- 必需索引
CREATE INDEX idx_user_wx_openid ON user(wx_openid);
CREATE INDEX idx_user_subscribe_time ON user(subscribe_time);
```

**优化说明**：
- 添加了微信用户的完整信息字段
- 增加了状态字段用于用户管理
- 添加了最后登录时间用于活跃度分析
- 添加了更新时间戳便于数据追踪
- 添加了必要的索引提升查询性能

## 2. 书目表优化 (`classic_book`)

```sql
CREATE TABLE classic_book (
  id              VARCHAR(32) PRIMARY KEY,
  title           VARCHAR(100) NOT NULL,
  subtitle        VARCHAR(200),      -- 副标题
  author          VARCHAR(100),      -- 作者
  dynasty         VARCHAR(50),       -- 朝代
  category        VARCHAR(50),       -- 分类
  description     TEXT,              -- 描述
  cover_image     TEXT,              -- 封面图片
  total_chapters  INT DEFAULT 0,     -- 章节数量
  total_sentences INT DEFAULT 0,     -- 总句数
  level           TINYINT DEFAULT 1, -- 难度等级 1-5
  is_published    BOOLEAN DEFAULT FALSE, -- 是否发布
  order_index     INT DEFAULT 0,
  created_at      TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at      TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);

-- 必需索引
CREATE INDEX idx_classic_book_category ON classic_book(category);
CREATE INDEX idx_classic_book_order ON classic_book(order_index);
CREATE INDEX idx_classic_book_level ON classic_book(level);
CREATE INDEX idx_classic_book_published ON classic_book(is_published);
```

## 3. 章节表优化 (`classic_chapter`)

```sql
CREATE TABLE classic_chapter (
  id              VARCHAR(32) PRIMARY KEY,
  book_id         VARCHAR(32) NOT NULL,
  title           VARCHAR(200) NOT NULL,
  description     TEXT,              -- 章节简介
  total_sentences INT DEFAULT 0,     -- 本章节句数
  order_index     INT NOT NULL DEFAULT 0,
  is_published    BOOLEAN DEFAULT FALSE,
  created_at      TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at      TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  FOREIGN KEY (book_id) REFERENCES classic_book(id) ON DELETE CASCADE
);

-- 必需索引
CREATE INDEX idx_classic_chapter_book ON classic_chapter(book_id);
CREATE INDEX idx_classic_chapter_order ON classic_chapter(book_id, order_index);
CREATE INDEX idx_classic_chapter_published ON classic_chapter(is_published);
```

## 4. 句子表优化 (`classic_sentence`) - 核心表

```sql
CREATE TABLE classic_sentence (
  id              VARCHAR(32) PRIMARY KEY,
  chapter_id      VARCHAR(32) NOT NULL,
  content         TEXT NOT NULL,      -- 原文内容
  pinyin          TEXT,               -- 拼音
  note            TEXT,               -- 注释
  translation     TEXT,               -- 译文（可选）
  audio_url       TEXT NOT NULL,      -- 音频文件地址
  audio_duration  INT,                -- 音频时长（秒）
  audio_file_size INT DEFAULT 0,      -- 音频文件大小（字节）
  audio_format    VARCHAR(10) DEFAULT 'mp3', -- 音频格式
  order_index     INT NOT NULL DEFAULT 0,
  is_published    BOOLEAN DEFAULT TRUE, -- 是否发布
  view_count      INT DEFAULT 0,      -- 查看次数
  favorite_count  INT DEFAULT 0,      -- 收藏次数
  created_at      TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at      TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  FOREIGN KEY (chapter_id) REFERENCES classic_chapter(id) ON DELETE CASCADE
);

-- 必需索引
CREATE INDEX idx_classic_sentence_chapter ON classic_sentence(chapter_id);
CREATE INDEX idx_classic_sentence_order ON classic_sentence(chapter_id, order_index);
CREATE INDEX idx_classic_sentence_published ON classic_sentence(is_published);
CREATE FULLTEXT INDEX idx_classic_sentence_content ON classic_sentence(content); -- 全文检索
```

## 5. 听读进度表优化 (`user_listen_progress`)

```sql
CREATE TABLE user_listen_progress (
  id             VARCHAR(32) PRIMARY KEY,
  user_id        VARCHAR(32) NOT NULL,
  book_id        VARCHAR(32) NOT NULL,
  chapter_id     VARCHAR(32) NOT NULL,
  sentence_id    VARCHAR(32) NOT NULL,
  progress_sec   INT DEFAULT 0,      -- 当前句播放到的秒数
  play_speed     DECIMAL(2,1) DEFAULT 1.0, -- 播放速度
  auto_scroll    BOOLEAN DEFAULT TRUE,      -- 是否自动滚动
  show_pinyin    BOOLEAN DEFAULT TRUE,      -- 是否显示拼音
  last_play_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP, -- 最后播放时间
  last_position_percent DECIMAL(5,2) DEFAULT 0.00, -- 在句子中的播放百分比
  created_at     TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at     TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  FOREIGN KEY (user_id) REFERENCES user(id) ON DELETE CASCADE,
  FOREIGN KEY (book_id) REFERENCES classic_book(id) ON DELETE CASCADE,
  FOREIGN KEY (chapter_id) REFERENCES classic_chapter(id) ON DELETE CASCADE,
  FOREIGN KEY (sentence_id) REFERENCES classic_sentence(id) ON DELETE CASCADE,
  UNIQUE KEY uk_user_book (user_id, book_id)
);

-- 必需索引
CREATE INDEX idx_user_listen_progress_user ON user_listen_progress(user_id);
CREATE INDEX idx_user_listen_progress_book ON user_listen_progress(book_id);
CREATE INDEX idx_user_listen_progress_last_time ON user_listen_progress(last_play_time);
```

## 6. 听读记录表优化 (`user_listen_record`)

```sql
CREATE TABLE user_listen_record (
  id             VARCHAR(32) PRIMARY KEY,
  user_id        VARCHAR(32) NOT NULL,
  book_id        VARCHAR(32) NOT NULL,
  chapter_id     VARCHAR(32),
  sentence_ids   TEXT,               -- 逗号分隔的句子ID列表，记录本次听读涉及的句子
  listen_date    DATE NOT NULL,
  duration_sec   INT DEFAULT 0,      -- 本次听读时长（秒）
  sentence_count INT DEFAULT 0,      -- 本次听读句子数量
  is_valid_day   BOOLEAN DEFAULT FALSE, -- 是否为有效听读日（>=180秒）
  created_at     TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at     TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  FOREIGN KEY (user_id) REFERENCES user(id) ON DELETE CASCADE,
  FOREIGN KEY (book_id) REFERENCES classic_book(id) ON DELETE CASCADE,
  FOREIGN KEY (chapter_id) REFERENCES classic_chapter(id) ON DELETE SET NULL,
  UNIQUE KEY uk_user_date (user_id, listen_date)
);

-- 必需索引
CREATE INDEX idx_user_listen_record_user ON user_listen_record(user_id);
CREATE INDEX idx_user_listen_record_date ON user_listen_record(listen_date);
CREATE INDEX idx_user_listen_record_book ON user_listen_record(book_id);
CREATE INDEX idx_user_listen_record_valid_day ON user_listen_record(is_valid_day);
```

## 9. 操作日志表（支持运营分析）

```sql
CREATE TABLE operation_log (
  id           VARCHAR(32) PRIMARY KEY,
  user_id      VARCHAR(32),
  operation    VARCHAR(50) NOT NULL,  -- 操作类型：play, pause, next, prev, favorite等
  target_type  VARCHAR(20) NOT NULL,  -- 目标类型：book, chapter, sentence
  target_id    VARCHAR(32) NOT NULL,  -- 目标ID
  extra_data   JSON,                  -- 额外数据，如播放时长、错误信息等
  ip_address   VARCHAR(45),           -- IP地址
  user_agent   TEXT,                  -- 用户代理
  created_at   TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (user_id) REFERENCES user(id) ON DELETE SET NULL
);

-- 必需索引
CREATE INDEX idx_operation_log_user ON operation_log(user_id);
CREATE INDEX idx_operation_log_operation ON operation_log(operation);
CREATE INDEX idx_operation_log_target ON operation_log(target_type, target_id);
CREATE INDEX idx_operation_log_time ON operation_log(created_at);
```

## 10. 核心功能支持的关键优化

### 断点续听优化
- `user_listen_progress`表存储了完整的播放设置（速度、滚动、拼音显示等）
- 增加了`last_position_percent`字段，更精确地记录播放位置
- 增加了`last_play_time`便于判断是否需要恢复播放

### 统计分析优化
- `user_listen_record`表增加了`is_valid_day`字段，便于计算连续听读天数
- 增加了`sentence_ids`字段，记录每次听读的具体句子
- 增加了`sentence_count`便于统计每日学习量

### 性能优化
- 为所有外键字段添加了索引
- 为常用的查询条件添加了复合索引
- 在句子表中添加了全文索引，支持内容搜索

### 音频管理优化
- 在句子表中增加了音频文件大小、格式等信息
- 便于客户端做音频预加载和缓存管理
- 支持不同质量的音频文件管理

这个优化后的数据库设计专注于支持初始版本的核心功能，移除了社交互动和多角色相关的复杂设计，确保结构简洁高效，同时为后续功能扩展预留了空间。所有设计都考虑了性能和可维护性，确保在高并发场景下仍能保持良好的响应速度。
