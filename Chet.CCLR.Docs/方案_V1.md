# 国学经典听读小程序 · 完整产品方案（听读核心版）

> 产品定位：**沉浸式国学听读工具**
> 核心体验：播放时句子高亮 + 自动翻页 + 断点续听
> 目标人群：儿童启蒙 + 成人静心阅读

---

# 一、产品核心原则

1. 听是主体验，读是辅助沉浸
2. 一句 = 一个播放单元
3. 播放必须顺滑稳定
4. 所有统计围绕“听读行为”

---

# 二、功能结构总览

## 1️⃣ 听读播放（核心）

功能：

- 一句一个音频
- 当前播放句子高亮
- 自动滚动到当前句
- 自动播放下一句
- 手动暂停 / 上一句 / 下一句
- 显示原文 + 拼音 + 简要注释

播放流程：

进入章节 → 加载句子列表 → 播放当前句

播放结束 → index++ → 高亮下一句 → 自动滚动 → 播放下一句

---

## 2️⃣ 断点续听

- 记录当前 book_id
- 记录当前 sentence_id
- 记录当前句播放到的秒数

下次进入自动恢复。

---

## 3️⃣ 每日推荐

系统每天推荐：

- 一小段经典（3~5 句）
- 支持直接播放

推荐规则（MVP）：

- 按顺序轮播
- 或固定经典池随机

---

## 4️⃣ 统计体系

统计内容：

- 连续听读天数
- 累计听读总时长
- 今日听读时长

统计规则：

- 当日听读 ≥ 3 分钟记为有效一天
- 连续天数由业务层计算

---

## 5️⃣ 收藏功能

- 可收藏单句
- 收藏列表可播放
- 支持取消收藏

---

# 三、页面结构设计

## 首页

内容：

- 今日推荐
- 最近在听（继续播放）
- 听读统计（连续天数 + 总时长）
- 书目列表

布局：

顶部：书名 + 章节

中部：

原文（大字）

拼音（小字，可开关）

当前句高亮

底部：播放控制栏

播放控制栏：

- 上一句
- 播放 / 暂停
- 下一句

交互细节：

- 当前句字体加粗或底色高亮
- 自动 scrollIntoView 平滑滚动
- 切换句子不闪屏

---

## 收藏页

- 列表展示收藏句
- 支持播放

---

## 个人中心

- 连续听读天数
- 累计听读时长
- 今日听读时长
- 听读历史

# 四、数据库设计

## 1️⃣ 用户表 user

```sql
CREATE TABLE user (
  id            TEXT PRIMARY KEY,
  wx_openid     TEXT NOT NULL UNIQUE,
  nickname      TEXT,
  avatar_url    TEXT,
  created_at    DATETIME NOT NULL
);
```

---

## 2️⃣ 书目表 classic_book

```sql
CREATE TABLE classic_book (
  id           TEXT PRIMARY KEY,
  title        TEXT NOT NULL,
  category     TEXT,
  description  TEXT,
  order_index  INTEGER
);
```

---

## 3️⃣ 章节表 classic_chapter

```sql
CREATE TABLE classic_chapter (
  id          TEXT PRIMARY KEY,
  book_id     TEXT NOT NULL,
  title       TEXT,
  order_index INTEGER,
  FOREIGN KEY(book_id) REFERENCES classic_book(id)
);
```

---

## 4️⃣ 句子表 classic_sentence（核心）

```sql
CREATE TABLE classic_sentence (
  id              TEXT PRIMARY KEY,
  chapter_id      TEXT NOT NULL,

  content         TEXT NOT NULL,
  pinyin          TEXT,
  note            TEXT,

  audio_url       TEXT NOT NULL,
  duration_sec    INTEGER,

  order_index     INTEGER,

  FOREIGN KEY(chapter_id) REFERENCES classic_chapter(id)
);
```

---

## 5️⃣ 听读进度表 user_listen_progress

```sql
CREATE TABLE user_listen_progress (
  user_id        TEXT NOT NULL,
  book_id        TEXT NOT NULL,

  sentence_id    TEXT NOT NULL,
  progress_sec   INTEGER,

  updated_at     DATETIME NOT NULL,

  PRIMARY KEY (user_id, book_id)
);
```

---

## 6️⃣ 听读记录表 user_listen_record

用于统计时长与连续天数。

```sql
CREATE TABLE user_listen_record (
  id             TEXT PRIMARY KEY,
  user_id        TEXT NOT NULL,
  book_id        TEXT NOT NULL,

  listen_date    DATE NOT NULL,
  duration_sec   INTEGER,

  created_at     DATETIME NOT NULL
);
```

---

## 7️⃣ 收藏表 user_favorite_sentence

```sql
CREATE TABLE user_favorite_sentence (
  user_id      TEXT NOT NULL,
  sentence_id  TEXT NOT NULL,
  created_at   DATETIME NOT NULL,

  PRIMARY KEY (user_id, sentence_id)
);
```

# 五、播放状态机设计（核心逻辑）

状态：

- idle
- playing
- paused
- ended

流程：

play()

↓

加载当前句 audio

↓

playing

↓

onEnded

↓

index++

↓

scrollIntoView

↓

继续播放

暂停：

- 记录 progress_sec
- 写入 user_listen_progress

退出页面：

- 同步当前句与秒数

---

# 六、统计逻辑

连续天数计算：

- 查询最近 N 天 listen_record
- 判断是否连续

累计时长：

- SUM(duration_sec)

今日是否有效：

- 今日 listen_duration ≥ 180 秒

# 八、听读页交互细节设计（核心体验拆解）

> 目标：让用户在播放过程中进入“沉浸状态”，操作感弱、节奏感强。

---

# 一、整体视觉结构

页面结构分三层：

① 顶部信息区（弱存在）

② 中部正文区（视觉核心）

③ 底部播放控制区（低干扰）

---

# 二、中部正文区设计（重点）

## 1️⃣ 排版结构

每个句子结构：

- 原文（大字，主视觉）
- 拼音（小字，灰色）
- 注释（可折叠，小号字体）

示例层级：

原文：24px~28px

拼音：14px~16px

注释：14px

行距建议：1.6~1.8

---

## 2️⃣ 当前播放句高亮策略

不要使用“刺眼高亮色”。

推荐三层变化：

① 字体颜色加深（#222 → #000）

② 轻微放大（scale 1.02）

③ 添加淡色背景块（极浅米色或灰色）

动画时间：200ms ease-out

避免：

- 闪烁
- 跳变
- 强对比荧光色

目标：自然被“推到前景”，而不是被“点亮”。

---

# 三、自动滚动设计（关键体验）

核心原则：

> 当前播放句始终位于屏幕中部略偏上位置

推荐策略：

播放新句时：

- 使用 scrollIntoView
- block: "center"
- behavior: "smooth"

如果句子较短：

- 保持整体缓慢移动
- 不要突然跳到顶部

滚动时机：

在 onEnded → index++ 后 50~100ms 执行滚动

避免滚动与音频加载同时触发造成卡顿。

---

# 四、播放控制区设计

布局：

[ 上一句 ]   [ 播放/暂停 ]   [ 下一句 ]

交互规则：

- 播放按钮为主按钮（更大）
- 上/下一句为次按钮
- 长按下一句可连续快进

按钮状态：

playing → 显示暂停图标

paused → 显示播放图标

---

# 五、拼音显示策略

默认开启。

支持：

右上角“拼”按钮开关

关闭时：

- 拼音区域高度平滑收起
- 过渡时间 150ms

避免页面抖动。

---

# 六、注释显示策略

默认折叠。

点击句子 → 展开当前句注释。

展开时：

- 仅当前句展开
- 其他句自动收起
- 展开动画 200ms

---

# 七、沉浸模式（建议加入）

进入方式：

双击正文区域。

效果：

- 顶部栏隐藏
- 底部控制区半透明
- 页面背景偏暖色

退出方式：

单击页面显示控制区。

---

# 八、播放状态与UI同步机制

状态：

- idle
- playing
- paused

UI响应规则：

playing：

- 当前句高亮
- 自动滚动

paused：

- 高亮保留
- 不滚动

切换句子时：

- 立即更新高亮
- 再触发滚动
- 最后加载音频

顺序不能错，否则会视觉跳动。

---

# 九、断点续听视觉反馈

如果恢复播放：

- 页面直接定位到上次句子
- 高亮
- 显示“已恢复播放”轻提示（1秒消失）

不要弹窗。

---

# 十、性能优化建议

1. 句子列表虚拟滚动（超过100句时）
2. 音频提前 preload 下一句
3. 避免每句独立 setData 过多更新

---

# 十一、用户体验目标

用户理想感受：

- 不需要操作
- 不会被打断
- 节奏自然
- 有被引导的感觉

如果用户“忘记自己在操作手机”，说明设计成功。
