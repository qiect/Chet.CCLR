# 道德经音频拆解完整解决方案

## 方案概述

本方案提供了一套完整的音频处理流程，将长音频文件拆分为与句子一一对应的音频片段。方案包括两个核心工具：

1. **AudioProcessor**：控制台应用，负责文本分句和音频切割
2. **AudioTagger**：WPF应用，负责音频时间轴标注

## 工具1：AudioProcessor（音频处理器）

### 功能特性
- 文本自动分句
- 音频批量切割
- 数据库脚本生成
- 命令行交互式操作

### 使用方法
```bash
cd Chet.CCLR.AudioProcessor\AudioProcessor
dotnet run
```

### 核心功能
1. **文本分句**
   - 使用正则表达式识别中文标点
   - 生成句子列表文件

2. **音频切割**
   - 读取时间轴标签文件
   - 调用FFmpeg进行音频切割
   - 生成独立的音频片段

3. **数据库集成**
   - 生成SQL插入脚本
   - 关联句子与音频文件
   - 支持批量导入

### 输出文件
```
output/
├── sentences.txt       # 分句结果
├── labels.txt          # 时间轴标签（需要用户生成）
├── audio/              # 音频片段目录
│   ├── 001.mp3
│   ├── 002.mp3
│   └── ...
├── insert_script.sql   # 数据库插入脚本
└── README.md          # 使用说明
```

## 工具2：AudioTagger（音频标签器）

### 功能特性
- 音频播放控制
- 标签添加/编辑/删除
- 实时时间显示
- 标签文件导出

### 使用方法
```bash
cd Chet.CCLR.AudioTagger\AudioTagger
dotnet run
# 或直接运行
AudioTagger.exe
```

### 核心功能
1. **音频播放**
   - 使用NAudio进行高质量音频播放
   - 实时显示播放进度
   - 支持暂停/继续

2. **标签管理**
   - 空格键快速添加标签
   - 标签列表显示和管理
   - 句子内容编辑

3. **数据导出**
   - 生成符合AudioProcessor格式的标签文件
   - 支持UTF-8编码

## 核心算法

### 文本分句算法
```csharp
// 使用正则表达式识别中文标点
var pattern = @"(?<=[。！？；])\s*";
var parts = Regex.Split(text, pattern)
                 .Select(x => x.Trim())
                 .Where(x => !string.IsNullOrWhiteSpace(x))
                 .ToList();
```

### 标签导出格式
```
开始时间\t结束时间\t序号
```

示例：
```
0.000000	5.230000	001
5.230000	10.450000	002
```

## 常见问题

### Q1: 为什么需要两个工具？
A: 分离职责，AudioTagger专注于高质量的音频标注，AudioProcessor专注于批量处理和数据库集成。

### Q2: AudioTagger和Audacity有什么区别？
A: AudioTagger是为本项目定制的，界面更简洁，操作更直观，且直接输出AudioProcessor所需的格式。

### Q3: 能否自动化处理？
A: 目前需要人工听音频打标签，未来可考虑集成Whisper等ASR工具实现自动对齐。

### Q4: 如何处理多章音频？
A: 每章单独处理，分别制作标签文件，编号从001开始。
