# 日志配置

## 1. 项目中的 Serilog 配置

### 1.1 配置结构

项目的 Serilog 配置位于 `appsettings.json` 文件中的 `Serilog` 节点：

```json
{
  "Serilog": {
    "Using": [ "Serilog.Sinks.Console", "Serilog.Sinks.File" ],
    "MinimumLevel": {
      "Default": "Debug",
      "Override": {
        "Microsoft": "Warning",
        "Microsoft.AspNetCore": "Warning"
      }
    },
    "WriteTo": [
      {
        "Name": "Console"
      },
      {
        "Name": "File",
        "Args": {
          "path": "logs/log-.txt",
          "rollingInterval": "Day",
          "rollOnFileSizeLimit": true,
          "formatter": "Serilog.Formatting.Compact.CompactJsonFormatter, Serilog.Formatting.Compact"
        }
      }
    ],
    "Enrich": [ "FromLogContext", "WithMachineName", "WithThreadId" ]
  }
}
```

### 1.2 配置加载方式

Serilog 配置通过以下方式加载：

1. **Program.cs 中的配置**：
   ```csharp
   builder.Host.UseSerilog((context, configuration) =>
       context.ConfigureSerilog(configuration)
   );
   ```

2. **SerilogExtensions.cs 中的实现**：
   ```csharp
   public static void ConfigureSerilog(this HostBuilderContext context, LoggerConfiguration configuration)
   {
       configuration.ReadFrom.Configuration(context.Configuration);
   }
   ```

## 2. 配置项详细说明

### 2.1 基础配置

| 配置项 | 说明 | 示例值 |
|--------|------|--------|
| `Using` | 指定使用的 Serilog 接收器（Sinks） | `["Serilog.Sinks.Console", "Serilog.Sinks.File"]` |
| `MinimumLevel.Default` | 默认日志级别 | `Debug` |
| `MinimumLevel.Override` | 针对特定命名空间的日志级别覆盖 | `{"Microsoft": "Warning", "Microsoft.AspNetCore": "Warning"}` |

### 2.2 输出配置（WriteTo）

#### 2.2.1 控制台输出

```json
{
  "Name": "Console"
}
```

#### 2.2.2 文件输出

```json
{
  "Name": "File",
  "Args": {
    "path": "logs/log-.txt",
    "rollingInterval": "Day",
    "rollOnFileSizeLimit": true,
    "formatter": "Serilog.Formatting.Compact.CompactJsonFormatter, Serilog.Formatting.Compact"
  }
}
```

| 参数 | 说明 | 示例值 |
|------|------|--------|
| `path` | 日志文件路径 | `logs/log-.txt` |
| `rollingInterval` | 日志滚动间隔 | `Day`（按天滚动） |
| `rollOnFileSizeLimit` | 是否按文件大小限制滚动 | `true` |
| `formatter` | 日志格式化器 | `Serilog.Formatting.Compact.CompactJsonFormatter` |

### 2.3 日志增强（Enrich）

| 增强器 | 说明 |
|--------|------|
| `FromLogContext` | 从日志上下文添加属性 |
| `WithMachineName` | 添加机器名称 |
| `WithThreadId` | 添加线程ID |

## 3. 日志级别说明

| 级别 | 数值 | 用途 |
|------|------|------|
| Verbose | 0 | 详细诊断信息 |
| Debug | 1 | 调试信息 |
| Information | 2 | 一般信息 |
| Warning | 3 | 警告信息 |
| Error | 4 | 错误信息 |
| Fatal | 5 | 致命错误 |

## 4. 实际应用示例

### 4.1 在控制器中使用

```csharp
[ApiController]
[Route("api/[controller]")]
public class UsersController : ControllerBase
{
    private readonly ILogger<UsersController> _logger;
    private readonly IUserService _userService;

    public UsersController(ILogger<UsersController> logger, IUserService userService)
    {
        _logger = logger;
        _userService = userService;
    }

    [HttpGet("{id}")]
    public async Task<ActionResult<UserDto>> GetUser(int id)
    {
        _logger.LogInformation("正在获取用户ID为 {UserId} 的信息", id);

        try
        {
            var user = await _userService.GetUserByIdAsync(id);
            
            if (user == null)
            {
                _logger.LogWarning("未找到ID为 {UserId} 的用户", id);
                return NotFound(new { message = "用户不存在" });
            }

            _logger.LogInformation("成功获取用户ID为 {UserId} 的信息", id);
            return Ok(user);
        }
        catch (Exception ex)
        {
            _logger.LogError(ex, "获取用户ID为 {UserId} 的信息时发生错误", id);
            throw;
        }
    }
}
```

### 4.2 在服务中使用

```csharp
public class UserService : IUserService
{
    private readonly ILogger<UserService> _logger;
    private readonly IUserRepository _userRepository;
    private readonly IMapper _mapper;

    public UserService(
        ILogger<UserService> logger,
        IUserRepository userRepository,
        IMapper mapper)
    {
        _logger = logger;
        _userRepository = userRepository;
        _mapper = mapper;
    }

    public async Task<UserDto?> GetUserByIdAsync(int id)
    {
        _logger.LogDebug("开始执行 GetUserByIdAsync 方法，用户ID: {UserId}", id);

        var user = await _userRepository.GetByIdAsync(id);

        if (user != null)
        {
            var userDto = _mapper.Map<UserDto>(user);
            _logger.LogDebug("成功映射用户数据，用户ID: {UserId}", id);
            return userDto;
        }

        _logger.LogWarning("在数据库中未找到ID为 {UserId} 的用户", id);
        return null;
    }

    public async Task<UserDto> CreateUserAsync(UserCreateDto createUserDto)
    {
        _logger.LogInformation("开始创建新用户，用户名: {UserName}", createUserDto.Name);

        // 验证用户是否已存在
        var existingUser = await _userRepository.GetByEmailAsync(createUserDto.Email);
        if (existingUser != null)
        {
            _logger.LogWarning("尝试创建已存在的用户，邮箱: {Email}", createUserDto.Email);
            throw new ArgumentException($"邮箱 {createUserDto.Email} 已被使用");
        }

        var user = _mapper.Map<User>(createUserDto);
        user.PasswordHash = BCrypt.Net.BCrypt.HashPassword(createUserDto.Password);

        var createdUser = await _userRepository.AddAsync(user);
        await _userRepository.SaveChangesAsync();

        var userDto = _mapper.Map<UserDto>(createdUser);
        
        _logger.LogInformation("成功创建新用户，用户ID: {UserId}，用户名: {UserName}", 
            createdUser.Id, createUserDto.Name);

        return userDto;
    }
}
```

## 5. 结构化日志记录最佳实践

### 5.1 使用占位符

```csharp
// 好的做法 - 使用占位符
_logger.LogInformation("用户 {UserId} 在 {Timestamp} 执行了 {Action} 操作", userId, timestamp, action);

// 避免的做法 - 字符串插值
_logger.LogInformation($"用户 {userId} 在 {timestamp} 执行了 {action} 操作");
```

### 5.2 记录异常信息

```csharp
try
{
    // 一些可能引发异常的操作
}
catch (Exception ex)
{
    _logger.LogError(ex, "执行操作时发生错误，用户ID: {UserId}，操作类型: {OperationType}", userId, operationType);
    throw; // 重新抛出异常
}
```

### 5.3 添加日志上下文

```csharp
using (_logger.BeginScope("Request={RequestId} User={UserId}", requestId, userId))
{
    // 在此作用域内的所有日志都会包含 RequestId 和 UserId
    _logger.LogInformation("处理请求开始");
    // ... 业务逻辑 ...
    _logger.LogInformation("处理请求结束");
}
```

## 7. 性能考虑

### 7.1 条件日志记录

```csharp
// 在性能敏感的代码中使用 IsEnabled 检查
if (_logger.IsEnabled(LogLevel.Debug))
{
    var expensiveValue = ExpensiveMethod();
    _logger.LogDebug("昂贵计算的结果: {ExpensiveValue}", expensiveValue);
}
```

### 7.2 异步日志记录

Serilog 默认是线程安全的，并且大多数 Sink 都支持异步写入，这有助于减少日志记录对应用程序性能的影响。

## 8. 高级配置选项

### 8.2 自定义日志格式

可以通过实现自定义的 `ITextFormatter` 来定义日志输出格式。

## 9. 故障排除

### 9.1 检查日志配置

如果日志没有按预期工作，可以启用自举日志来诊断问题：

```csharp
Log.Logger = new LoggerConfiguration()
    .WriteTo.Console()
    .CreateBootstrapLogger();

Log.Information("Starting up");
```

### 9.2 验证配置文件

确保配置文件语法正确，并且 Serilog 相关的 NuGet 包已正确安装：

- `Serilog.AspNetCore`
- `Serilog.Settings.Configuration`
- `Serilog.Sinks.Console`
- `Serilog.Sinks.File`
- `Serilog.Enrichers.Environment`
- `Serilog.Enrichers.Thread`

通过合理配置和使用 Serilog，您可以获得强大的日志记录能力，帮助您更好地监控和调试应用程序。
